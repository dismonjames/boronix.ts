# Release v0.5.0 - Production Build Hardening

## Highlights

- **Production environment validation**: Added checks for port (1-65535), host, runtime compatibility, and directory path traversal.
- **Session secret hardening**: Suppresses duplicate development session secret warnings, and enforces session secrets in production when sessions are utilized.
- **Version 1 build manifest**: Standardized `.boronix/manifest.json` format and centralized its parsing and validation.
- **Atomic build output writes**: Uses `.boronix.tmp` to prevent corrupted build folders, moving it to `.boronix` atomically on success.
- **Hardened production start**: `boronix start` now reads the build manifest and fails if it is missing, invalid, or built for a different runtime.
- **Production-safe error responses**: Stack traces, file paths, configurations, and environment variables are no longer leaked on 5xx page errors in production. API errors return sanitized JSON with Request IDs.
- **Request IDs**: Generates Request IDs (`requestId` / `x-boronix-request-id`) for tracing.
- **Static Cache & Security headers**: In production, static files receive appropriate `Cache-Control`, `ETag`, and `Last-Modified` headers. Security headers (`X-Content-Type-Options: nosniff`, `Referrer-Policy`, etc.) are served by default.
- **Graceful Shutdown**: Server captures SIGINT/SIGTERM, stops accepting requests, drains existing connections (up to 5s), and exits code 0.
- **Optional Health Endpoint**: Added configurable `/health` route.
- **Doctor Production check**: `boronix doctor --production` runs production-readiness checks.

## Production Behavior Changes & Migration Note

1. `boronix start` now strictly requires a build manifest generated by `boronix build`.
2. In production, if session or auth features are detected, the app will refuse to boot or build unless `session.secret` is set in configuration or `BORONIX_SESSION_SECRET` is provided in environment variables.
3. Legacy manifests will raise `KQ_BUILD_VERSION_UNSUPPORTED` and ask to rerun the build command.

## Build and Start Commands

```bash
# Build production assets
bun run build

# Start production server
BORONIX_SESSION_SECRET="your-long-random-secret" bun run start
```

## Health Endpoint Config

```ts
export default defineConfig({
  health: {
    enabled: true,
    path: "/health"
  }
})
```

## Doctor --production

```bash
bunx boronix doctor --production
```

## Known Limitations

- Atomic folder rename relies on the underlying filesystem. For cross-device link errors, it falls back to standard rename/move operations.

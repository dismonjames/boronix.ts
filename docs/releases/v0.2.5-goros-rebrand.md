# Boronix v0.2.5 Release Notes

This milestone release rebrands the entire framework from **Kumquat.ts** to **Boronix** and prepares package tarballs for npm release.

## Brand Modifications

- **Core Package**: Rebranded from `kumquat` to `boronix`.
- **Creator Package**: Rebranded from `create-kumquat` to `create-boronix`.
- **CLI Utility**: Rebranded binary and executions from `kumquat` to `boronix`.
- **User Imports**: All framework features are now imported via `boronix` instead of `kumquat`.
- **Configuration File**: Changed the primary workspace configuration filename to `boronix.config.ts`.
- **Build Output**: Re-routed production manifest compilations to `.boronix/` instead of `.kumquat/`.

## Deprecation Warnings

- If `kumquat.config.ts` is detected in the root directory, Boronix will print a deprecation warning guiding users to rename it to `boronix.config.ts`.
- If an old `.kumquat/` manifest is found, Boronix will fail execution guiding users to rebuild the manifest under `.boronix/`.
